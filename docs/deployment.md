# Развертывание

Проект полностью контейнеризирован с помощью Docker и Docker Compose, что упрощает его развертывание как для локальной разработки, так и для продакшен-среды.

## Требования

- **Docker**
- **Docker Compose**

## Локальная разработка

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/d1sxrder3d/media_service_py
    cd media_service_py
    ```

2.  **Создайте файл конфигурации:**
    Скопируйте файл `.env.example` в `.env`. Для локального запуска значения по умолчанию подходят.
    ```bash
    cp .env.example .env
    ```

3.  **Запустите проект с помощью Docker Compose:**
    ```bash
    docker-compose up --build
    ```
    Эта команда соберет образы и запустит все сервисы, включая базу данных, Redis и MinIO. API будет доступен по адресу `http://localhost:8000`.

## Запуск в продакшене

Для запуска в продакшен-среде используется файл `docker-compose.prod.yml`. Он расширяет базовую конфигурацию и не включает в себя сервисы `db`, `redis` и `minio`, так как предполагается, что они будут предоставлены как внешние управляемые сервисы (например, Amazon RDS, ElastiCache, S3).

1.  **Подготовьте переменные окружения.** Создайте `.env` файл на сервере и укажите в нем реальные адреса и учетные данные для подключения к вашей базе данных, брокеру сообщений и S3-хранилищу.

2.  **Запустите приложение:**
    ```bash
    docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
    ```
